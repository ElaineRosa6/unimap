## 本轮修复内容摘要

### P0 — 安全阻断（2 项）

1. **Webhook SSRF DNS/重定向绕过** — `ValidateWebhookURLPublic` 增加 DNS 解析校验所有返回 IP；`sendWebhookNotification` 改用 `safeWebhookClient()`：拒绝重定向 + 安全 DialContext 阻止私有 IP
2. **APIKeyManager 持久化后密钥不可验证** — 引入 `KeyHash` 字段（SHA-256）；`GenerateAPIKey` 改用 ID 为 map key；`ValidateAPIKey`/`UpdateLastUsed` 改用 hash 比对；`expiresIn=0` 表示永不过期

### P1 — 可靠性（3 项）

3. **分布式任务队列持久化** — 新增 `NewTaskQueueWithPath()` + JSON 快照持久化，所有状态变更调用 `saveLocked()`
4. **auth 平行认证接入** — Server 结构体新增 `apiAuth` 字段；路由层接入 `OptionalAPIKey` 中间件
5. **备份错误被吞掉** — source 失败累积错误返回；tar 写入失败累积并在最终返回部分成功错误

### P2 — 次要问题（5 项）

6. **BackupConfig 注释不一致** — 注释改为"绝对路径或相对路径"
7. **限流 X-Real-IP 无条件信任** — 加入 `isPrivateOrInternalHost` 代理检查
8. **Admin Token 非loopback 静默生成** — 打印 token 并提示保存
9. **CLI JSON 输出文件覆盖** — `writeJSONFile` 改用 `O_EXCL`
10. **CORS config 层默认值缺 X-Admin-Token** — 添加至 config 默认值

### 附加修复

- `TestGenerateAPIKey/zero_expiration` — `expiresIn=0` 返回 `IsZero` 时间；测试逻辑更新
- `TestUpdateLastUsed` — 改为按 ID 查找验证
